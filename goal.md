## M2 — Single-GPU GPT-2 forward (🟡 partial)

What is done:

- [x] `llmc/tk/gemm_h100.cuh` — TK bf16 H100 GEMM ported into header form. Specialisations cover both `A*B` and `A*B^T`; the `A*B^T` path is used for llm.c model weights stored as `(OC, C)`. Two N-block sizes are exposed: `matmul_default<2,4,8>` (N % 256, default) and `matmul_small_n<2,2,8>` (N % 128, used by the GPT-2 LM head). Persistent grid (132 SMs), TMA producer, WGMMA consumer.
- [x] `llmc/matmul.cuh` — C-style `matmul_forward(out, inp, weight, bias, B, T, C, OC, stream)`. Dispatches to the TK `A*B^T` GEMM based on `OC % 256`, matching llm.c's `(OC, C)` weight layout. The default path applies bias in a separate pass (`add_bias_kernel`); the GPT-2 MLP up-projection can opt into the compile-wired TK finish-path bias+GELU epilogue through `matmul_forward_gelu` / `-ge 1`, pending H100 numerical validation.
- [x] `llmc/matmul.cuh` — `matmul_backward_bias_kernel9` and `reduce_add_sum_kernel` ported verbatim from llm.c. They are the column-reduction kernels the M3 backward will call.
- [x] `llmc/tk/attention_h100.cuh` + `llmc/attention.cuh` — TK MHA forward fast path for `T % 192 == 0`, plus padded TK dispatch for other sequence lengths (including GPT-2 `T=1024`). Keeps the verbatim llm.c QKV permute/unpermute glue.
- [x] `llmc/layernorm.cuh` — llm.c LayerNorm / fused residual / backward correctness baseline ported so the GPT-2 trainer API compiles. Forward and fused-residual forward now dispatch to the TK fork for supported widths; CUDA baseline remains as fallback.
- [x] `llmc/tk/layernorm_tk.cuh` — TK LayerNorm forward fork compile-wired for widths `{768, 1024, 1280, 1600, 2048, 4096}`. Dropout stripped, internal sync removed, `cudaStream_t` launchers added, and `mean`/`rstd` saved for GPT-2 backward.

## M3 — Single-GPU GPT-2 backward (🟡 partial)

- [x] **TK MHA backward** wrapper in `llmc/attention.cuh` + `llmc/tk/attention_h100.cuh`. Ported `bwd_attend_prep_ker` + `bwd_attend_ker` from `ThunderKittens/kernels/attention/mha_h100/mha_h100.cu`. The fast path supports `head_dim ∈ {64, 128}` and `T % 256 == 0`, uses the forward-saved LSE in `att`, and falls back to the slow CUDA recompute baseline for unsupported shapes. `dev/cuda/test_attention.cu` compile-wires a CPU-reference forward/backward smoke harness; H100 numerical parity still pending.
- [x] **Hand-written LayerNorm backward** in `llmc/layernorm.cuh`. `layernorm_backward_kernel10` keeps the llm.c cross-block atomic-counter accumulator pattern and uses a TK `kittens::warp::sum` shared-vector helper for the row-wise backward reductions. `dev/cuda/test_layernorm.cu` compile-wires a CPU-reference forward/fused/backward smoke harness; H100 numerical parity still pending.
- [x] **TK `matmul_backward`** in `llmc/matmul.cuh`.
  - `dInp = dOut · W` is wired through the existing TK `A*B` GEMM because `W` is stored `(OC, C)`.
  - `dWeight = dOutᵀ · Inp` uses TK `A^T*B` when the destination is known to be zero (`micro_step == 0`).
  - Accumulated `dWeight += dOutᵀ · Inp` uses a dedicated aligned `matmul_scratch` activation buffer for the TK product, then adds the temporary result into the gradient tensor. The slow CUDA `+=` kernel remains only as a fallback for unsupported shapes or missing scratch.
  - `dBias = column-reduction of dOut` uses `matmul_backward_bias_kernel9` + `reduce_add_sum_kernel` when the auxiliary buffer is available.
- [x] **AdamW step** wired through (`llmc/adamw.cuh` already has the kernel; just call it from `train_gpt2.cu`).
- [x] **`global_norm` grad-clip** wired through.
- [x] **`test_gpt2.cu`** ported from `llm.c/test_gpt2.cu` and compile-checked. Loads `gpt2_124M_debug_state.bin`, runs forward + 10 AdamW steps, compares against the PyTorch reference.

## Top risks (kept honest)

1. **Custom GQA + RoPE-fused attention (M6)** — no full upstream TK reference. Forward and supported-shape backward are compile-wired; TK-supported shapes now rotate Q/K in the tile-load path, while fallback shapes use fused materialization/unpermute. H100 numerical validation is still pending.
2. **GPT-2 backward runtime parity is compile-wired but unverified** — TK MHA backward and LayerNorm backward both compile, and `test_attention`/`test_layernorm` cover focused GPT kernel shapes at compile time, but MHA reduction order and LayerNorm partial-sum geometry still need real H100 parity runs.
3. **Numerical-parity tolerances** — re-derived per tensor; expect ~10× looser than llm.c on attention bwd. Document in [`docs/testing.md`](docs/testing.md).
4. **TK MHA `head_dim ∈ {64, 128}`** — adequate for GPT-2 (64) and Llama-3 (128). Any future config (HS=80, 96) needs a new TK specialization.
5. **Llama-3 8B fits poorly on 8×H100-80GB** — requires ZeRO-2 or 2-node ZeRO-1. v1's single-node Llama target is 1B; 8B is multi-node only.

## Out of scope for v1

- FP16 and FP32 activations (TK H100 GEMM/MHA are bf16-only; re-introducing them is v2 work).
- A100 / RTX 4090 backends, plus optimized B200/GB200/RTX 5090 TK model kernels. Blackwell builds are supported through CUDA correctness fallbacks for Hopper-only kernels.
- FP8 / int8 / mxfp8 (TK ships H100 kernels for these; this is the more interesting v2 story than FP16/FP32).
